COMBAT PROCEDURE NOTES


Generalized Sequence

  • Pre-combat set up

  • Combatant takes an action

    • Attack, Maneuver, or Use Item

  • After action checks

    • Win/Loss, Action counter/Turn swapper

  • Will need to make some corrections/changes as game develops

  • Some tweaks (specifically to Roll() numbers) for balancing

    -Written to resemble code



    Notes on formatting

    • Roll(f,n,b)

      • f = # of faces on die

      • n = # of die rolled

      • b = modifier (bonus) to roll


This is a variation on dice notation. For example: 1d20+2, one 20-sided die is rolled and 2 is added to the result.



Combat Sequence

  1. Initial Setup

    1. Turn order determination

      1. Player roll(20,1,b) → P

      2. Game roll(20,1,b) → G

      3. If (P > G)

        1. Then (T → 0, Goto [Action Menu])

        2. Else (T → 2, Goto [Game Actions])

    2. Encounter distance

      1. D → 3


        Is rolling for initiative in D&D Bonus tied to some undefined attribute, currently is 0

        Bigger value goes first

        T is turn tracker, see Turn Check notes


        Sets initial distance to 3



  2. Actions Menu

    1. Attack

      1. Cannon shot – Goto [Attacks (a)]

      2. Rifle shot – Goto [Attacks (b)]

      3. Boarding Party – Goto [Attacks (c)]

    2. Maneuver

      1. Close – Goto [Moves (a)]

      2. Gain – Goto [Moves (b)]

      3. Brace – Goto [Moves (c)]

    3. Use Item

      1. Shoring – Goto [Items (a)]

      2. Dressing – Goto [Items (b)]

      3. Switch Ammo – Goto [Items (c)]


        Enabled when D > 0

        Long range, enabled at D > 1 Enabled at D = 0


        D reduces by 1 D increases by 1

        Reduced crew casualties


        Recovers some Ship HP Recovers some crew

        Changes ammo type in Cannon



  3. After Action

    1. Distance

      1. If (D = 6)

        1. Goto [End States (a)]


          When D = 6, flee from battle

    2. Player Win checks

      1. Or (GH < 1, GC < 1)

        1. Goto [End States (b)]

    3. Player Loss checks

      1. Or (PH < 1, PC < 1)

        1. Goto [End States (c)]


          GH = Game HP, GC = Game crew


          PH = Player HP, PC = Player crew

    4. Turn Check

      1. 1 + T → T

      2. If (T = 4)

        1. Then (T → 0)

      3. If (T < 2)

        1. Then (Goto [Action Menu])

        2. Else (Goto [Game Actions])


          Ticks up turn tracker Tracker reset

          Player’s turn if T = 0 or 1 Game’s turn if T = 2 or 3



  4. End States

    1. Flee from battle

      1. Display “You lose sight of them”

      2. Return to [Travel Screen]

    2. Win battle

      1. Display “Success! You gained:”

      2. Display [loot gained]

      3. Return to [Travel Screen]

    3. Lose battle

      1. Display “You’ve been Sunk!”

      2. Display “GAME OVER”

      3. Return to [Title Screen]


        Default when out of battle


        Loot specific to enemy, need loot tables


        Restart or quit



  5. Attacks

    1. Cannon

      1. Distance Check

        1. If (D < 1)

          1. Then (Display “Too close”)

      2. Ammo Check

        1. And (AT = 1, A1 > 0)

          1. Then (Display “Out of shot”, Goto [Attack Menu])

        2. And (AT = 2, A2 > 0)

          1. Then (Display “Out of chain”, Goto [Attack Menu])


            Can’t use cannon if ship is next to the other ship


            AT is for ammo type

            A1 tracks ammo type one: ‘standard balls’

            A2 tracks ammo type two: ‘grape shot’

      3. Damage Roll

        1. If (AT = 1)

          i. GH - Roll(10,2,-D) → GH

          1. Then (…)


            1. GC – Roll(6,1,-D) → GC

            2. A1 – 1 → A1

        2. If (AT = 2)

          1. Then (…)

            1. GH – Roll(6,1,-D) → GH


              Normal shot damages the ship mostly with some loss of crew Distance impacts effectiveness


              Decrement’s ammo

              Grape shot mostly kills crew, doing little damage to ship


      4. Goto [After Action]

    2. Rifle

            1. GC – Roll(8,1,-D) → GC

            2. A2 – 1 → A2

              1. Distance Check

                1. If (D < 1)

                  1. Then (“Too close to shoot”, Goto [Attack Menu])

                    Rifles are for long range accuracy at the expense of damage

              2. Ammo Check

                1. If (A3 < 1)

                  1. Then (“Out of ammo!”, Goto [Attack Menu])

              3. Damage Roll

                1. GH – Roll(10,1,-D) → GH

                2. GC – Roll(4,1,0) → GC

                3. A3 – 1 → A3

              4. Goto [After Action]

    3. Boarding Party

      1. Distance Check

        1. If (D =/= 0)

          1. Then (“Out of range!”, Goto [Attack Menu]


            Boarding relies on crew numbers to deal damage


            Will need to be modified for combat without enemy crews

      2. Damage Roll

        1. GC – Roll(PC,1,b) → GC

        2. PC – Roll(20,1,b) → PC

      3. Goto [After Action]


        Damage based on crew pop. Some casualties occur


  6. Moves

    1. Close

      1. D – 1 → D

      2. Goto [After Action]

    2. Gain

      1. D + 1 → D

      2. Goto [After Action]

    3. Brace

      1. [This is to be temporary bonus to reduce crew loss during the game’s next turn]

      2. Goto [After Action]


        Probably implemented with a negative bonus on Game’s attack



  7. Items

    1. Shoring

      1. If (IS > 0) (Shoring Inventory)

        1. Then (Roll(8,3,b) + PH → PH

        2. Else (“None left to use!”)

      2. Goto [After Action]

    2. Dressings

      1. If (ID > 0) (Dressing Inventory)

        1. Then (Roll(10,1,b) + PC → PC


          Shoring is wooden bracing and looks like the inside of a mine shaft when implemented


          Dressing as in wound dressings

          1. Else (“None left to use!”)

      2. Goto [After Action]

        1. Switch Ammo

          1. If (AT = 1)

            1. Then (AT → 2)

            2. Else (AT → 1)

          2. Goto [After Action]


  8. Game Actions

    1. [Game actions will be similar to how Player actions work, but need to be written to specific enemies. A pirate ship for example will have access to the same attacks, items and maneuvers. But a sea monster will have very different options to combat the player, such as tentacle attacks, etc.]

    2. Goto [After Action]
